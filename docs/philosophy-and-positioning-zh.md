# Zephyr v0.3.1：保留主體性的自動化

捕捉想法應該輕鬆；承諾則必須清楚可見。Zephyr 將兩者分開：未確認內容在 `Capture/`，長期知識在 `Brain/`，明確選擇推進的工作在 `Active/`，歷史紀錄在 `Archive/`。

自動化應減少機械摩擦，而不是藏起決策。Zephyr 可自動觀察客觀訊號並建立注意力佇列；語意代理只有在明確啟用後才能建立獨立草稿。承諾、文字修改與生命週期移動仍需當下的人類批准。

Markdown 仍是真實紀錄，生成狀態集中在 `System/`，命令如實回報失敗，也沒有隱藏的雲端 fallback。詳見 [協定](../System/PROTOCOL.md) 與 [自動化模型](../System/AUTOMATION.md)。

代理的身分屬於外部工具，而非 Zephyr 核心。每次任務可由一個代理操作；其他代理只是可選、暫時的審閱者或專家，絕非永久階級。
